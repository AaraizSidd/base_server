🎯 Overview

The TODO API is a comprehensive task management system built using Express.js with TypeScript and MongoDB. It allows authenticated users to create, read, update, and delete their personal todos with advanced features.

Key Features

  • User authentication (JWT-based)
  • Role-based access control
  • Todo ownership verification
  • Advanced filtering and pagination
  • Status tracking (pending, in_progress, completed)
  • Priority levels (low, medium, high)
  • Due date management
  • Rate limiting on write operations
  • Comprehensive error handling

💾 Database Schema

Todo Collection Structure

{
  _id: ObjectId,              // Unique identifier (auto-generated)
  title: String,              // Required, 3-255 chars
  description: String,        // Optional, max 1000 chars
  status: String,             // pending | in_progress | completed
  priority: String,           // low | medium | high (default: medium)
  dueDate: Date,              // Optional expiry date
  userId: String,             // Reference to user who owns this todo
  createdAt: Date,            // Timestamp of creation
  updatedAt: Date             // Timestamp of last update
}
Indexes: userId is indexed for fast user-specific queries

🔌 API Endpoints

1. Create Todo

POST /v1/todo

Authentication: Required | Rate Limiting: Applied

Request Body:

{
  "title": "Buy groceries",
  "description": "Milk, eggs, bread",
  "priority": "high",
  "dueDate": "2026-03-15"
}

Response (201 Created):

{
  "success": true,
  "statusCode": 201,
  "message": "Operation is completed",
  "data": {
    "success": true,
    "data": {
      "_id": "65f1234...",
      "title": "Buy groceries",
      "status": "pending"
    }
  }
}

2. Get All Todos

GET /v1/todo

Query Parameters:

  • status: Filter by status (pending, in_progress, completed)
  • priority: Filter by priority (low, medium, high)
  • page: Page number (default: 1)
  • limit: Items per page (default: 10, max: 100)

Example: GET /v1/todo?status=pending&priority=high&page=1&limit=10

3. Get Specific Todo

GET /v1/todo/:id

Ownership Check: Yes | Returns: Single todo object

4. Update Todo

PUT /v1/todo/:id

All fields optional. Example request:

{
  "title": "Updated title",
  "status": "in_progress",
  "priority": "medium"
}

5. Mark Todo Complete

PATCH /v1/todo/:id/complete

Sets status to completed

6. Delete Todo

DELETE /v1/todo/:id

Ownership Check: Yes | Returns: Deleted todo object

🔐 Authentication & Authorization

Authentication Flow

┌────────────────────────────────┐ │ 1. User Registers & Logs In │ │ - Credentials validated │ │ - JWT tokens generated │ │ - Stored in cookies │ └──────────────┬─────────────────┘ │ ┌──────────────▼─────────────────┐ │ 2. Request to Todo Endpoint │ │ - Cookies sent with request │ │ - accessToken extracted │ └──────────────┬─────────────────┘ │ ┌──────────────▼─────────────────┐ │ 3. Authenticate Middleware │ │ - Verifies JWT token │ │ - Extracts userId │ │ - Attaches user to request │ └──────────────┬─────────────────┘ │ ┌──────────────▼─────────────────┐ │ 4. Controller Receives Req │ │ - req.authenticatedUser set │ │ - User ID available for ops │ └────────────────────────────────┘

Authorization (Ownership Verification)

For GET, PUT, DELETE, PATCH operations:

// In validation/validations.ts
todoOwnershipVerification: async (todoId, userId) => {
  // 1. Fetch todo from database
  const todo = await query.getTodoById(todoId)
  
  // 2. Check if todo exists
  if (!todo) throw new Error('Todo not found')
  
  // 3. Compare todo owner with authenticated user
  if (todo.userId.toString() !== userId.toString()) {
    throw new Error('Unauthorized')
  }
  
  // 4. Return todo if authorized
  return todo
}
Important: Users can only access/modify their own todos. The system verifies ownership on all operations.

✅ Validation & Error Handling

Error Handling Hierarchy

Error Type Status Code Example
Validation Error 422 Invalid input format, missing fields
Authentication Error 401 Missing or invalid token
Authorization Error 403 User accessing another user's todo
Not Found 404 Todo doesn't exist

📤 Curl Commands

1. Register User

curl -X POST http://localhost:3000/v1/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "phoneNumber": "12125551234",
    "password": "Secure@123"
  }'

2. Login

curl -X POST http://localhost:3000/v1/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{
    "email": "john@example.com",
    "password": "Secure@123"
  }'

3. Create Todo

curl -X POST http://localhost:3000/v1/todo \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "title": "Buy groceries",
    "description": "Milk, eggs, bread",
    "priority": "high",
    "dueDate": "2026-03-15"
  }'

4. Get All Todos

curl -X GET "http://localhost:3000/v1/todo?status=pending&priority=high&page=1&limit=10" \
  -H "Content-Type: application/json" \
  -b cookies.txt

5. Get Specific Todo

curl -X GET http://localhost:3000/v1/todo/{todoId} \
  -H "Content-Type: application/json" \
  -b cookies.txt

6. Update Todo

curl -X PUT http://localhost:3000/v1/todo/{todoId} \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "status": "in_progress",
    "priority": "medium"
  }'

7. Mark Todo Complete

curl -X PATCH http://localhost:3000/v1/todo/{todoId}/complete \
  -H "Content-Type: application/json" \
  -b cookies.txt

8. Delete Todo

curl -X DELETE http://localhost:3000/v1/todo/{todoId} \
  -H "Content-Type: application/json" \
  -b cookies.txt
